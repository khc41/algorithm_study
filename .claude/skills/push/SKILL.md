---

사용자가 커밋 메시지를 제공하면 다음 순서로 실행합니다:

## 절차

1. `git status`로 변경된 파일 목록 확인
2. 변경 파일이 없으면 사용자에게 알리고 종료
3. 변경 파일이 있으면 목록을 보여주고 계속 진행 (사용자가 이미 push를 요청했으므로 별도 확인 불필요)
4. `git add` — 새로 생성된 `.kt` 파일과 수정된 파일을 스테이징 (`.idea/`, `.vs/` 등 IDE 메타 파일은 제외)
5. 커밋 메시지로 커밋
6. 현재 브랜치를 원격(`origin`)에 푸시

## 커밋 메시지 규칙

사용자가 메시지를 주면 그대로 사용합니다. 메시지가 없으면 추가된 파일 이름 기반으로 자동 생성합니다:
- 예: `add two-sum solution in Kotlin`
- 예: `add 두큰수의합 solution in Kotlin`

## 실행 명령

```bash
git add <변경파일들>
git commit -m "<커밋메시지>"
git push origin <현재브랜치>
```

## 완료 후

푸시 성공 여부와 원격 브랜치 URL을 알려줍니다.
